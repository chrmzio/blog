Dotfiles manager Dotty

Started to work on my custom dotfiles manager. I’ve learned I very often appreciate imperfect custom software based on my current workflow and knowledge rather than full featured applications which I repeatedly need to learn how to use. It just starts from little scripts and ends with anything I am capable of creating myself.

And this could also be a script probably, but whatever, I am learning Go, so why not writing it in Go.

Dotty

So it will look something like this:

Dotty manages your dotfiles by syncing them with a Git repository.

Configuration file is located at ~/.config/dotty/config.toml and contains:
  - repo_url: URL to your GitHub repository
  - dotfiles: List of file paths to manage

Example configuration:
  repo_url = "https://github.com/username/dotfiles"
  dotfiles = [
    "~/.bashrc",
    "~/.vimrc",
    "~/.config/nvim/init.vim"
  ]

Usage:
  dotty [flags]
  dotty [command]

Available Commands:
  add         Add dotfiles to manage
  completion  Generate the autocompletion script for the specified shell
  help        Help about any command
  init        Initialize dotty configuration
  remove      Remove dotfiles from management
  status      Show status of managed dotfiles
  sync        Sync dotfiles with repository

Flags:
  -c, --config string   config file (default is $HOME/.config/dotty/config.toml)
  -h, --help            help for dotty
  -v, --verbose         verbose output

Use "dotty [command] --help" for more information about a command.

Not everything is implemented yet. You can check the repo, it’s listed in Projects.
